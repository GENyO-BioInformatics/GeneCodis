GeneCodis is a web-based tool for the ontological analysis of lists of genes. It can be used to determine biological annotations or combinations of annotations that are significantly associated to a list of genes under study with respect to a reference list. As well as single annotations, this tool allows users to simultaneously evaluate annotations from different sources, for example GO Biological Process and KEGG. To this end, and before computing p-values, it uses the apriori algorithm to extract sets of annotations that frequently co-occur in the analyzed list of genes. More information can be found in Carmona-Saez et al. (BMC Bioinformatics, 2006).

GeneCodis supports different gene nomenclatures and other types of input which are, micro-RNAs and CPGs. Genes can be set to act as transcription factors, which along with an input of miRNAs and CPGs, will be converted to its genes target or, in case of the CPGs, those where they occur. For all organisms, users can input entrez, symbol, ensembl and uniprot ids, as well as synonyms symbols collected in the NCBI FTP gene_info.gz file. CPGs allowed derive from EPIC i.e.: cg00050873 and are extracted from Bioconductor's package (IlluminaHumanMethylationEPICanno.ilm10b4.hg19). The miRNAs allowed, i.e: hsa-miR-122-5p, come from miRTarBase.

1º Choose an organism

Select one of the fifteen possible organisms listed in this page SECTION "Organisms".

2º Choose the type of input

By default GeneCodis will treat your input as genes but it can be treated as TFs (Transcription Factors). TFs are only available for Humans. If this type of input is selected, genes will be transformed to their targets which also happens if your input is a list miRNAs or CPGs but in these cases it will be identified automatically.

3º Paste your query list/s

Genes, Transcription Factors, CPGs and/or microRNAs in any of the allowed nomenclatures, as explained in "Allowed Input" section of this page.

4º Choose the annotation/s

At least one must be selected. The available annotations vary for each organism, check "Organisms" help.

5º Advanced Options

Test

Hypergeometric by default, also Chi-Square test is available. See details at "Stats".

pValue Correction

FDR by default. You can choose Permutations test or None to obtain only pValues not corrected. See details at "Stats".

Mode of analysis

Single by default. It is also possible Comparative, which analyse the common elements and uniques of the two input lists. If this last option is selected, you will be able to name each list, if not they are called "input1" and "input 2", respectively.

Min genes annotated

Annotations or combinations of annotations that do not have at least the selected number of genes will not be shown. This parameter is only relevant for the co-ocurrence analysis, the singular analyses are always computed with a minimum support of 1.

Universe

By default, all the genes of the organism that have one annotation associated. It will differ in each annotation data base.

Email

Introduce your email so that we can send you the link with results. If you do not want to submit it please be sure to copy the report url to recover your job anytime.

Report Name

Personalise your analysis with a name.

GeneCodis reports only significant results, pValue below 0.05, which in the website are limited so that only the top 50 terms are shown. If you want to see all the terms enriched, click in "Download Full Table".

Table

There is one for every selected annotation database, pluse one for the CoAnnotation analysis. The table is linked to two visualizations "Gene-Clustered" and "Bar-Chart", the terms selected in the table will be remarked in these. The columns of the table vary according to the statistical test/s chosen and its pValue correction method. The different columns are:

Annotations

The standarized code that identifies each term. It is specific to each ontology.

Terms

The human readable names of the annotations.

Hyp/Chi pVal/pValAdj

Hyp refers to the Hypergeometric test, and Chi to the Chi-Square test. You can apply one or both at the same time. The number of these columns is the pValue resulting from applying the test/s selected. Additionally, another column with the adjusted pValues by Permutations or False Discovery Rate (FDR), might appear always that a "pValue Correction" method is selected.

Term's Genes

Number of genes that are annotated to each annotation.

Term's Genes Found

Number of genes in the input list that part of the annotation.

Genes Universe

The number of genes that are annotated for the selected database/s and organism but, optionally, it will match the valid input in "Advanced Options > Universe > Custom".

Genes

The input genes associated to each annotation separated by comma.

Gene-Clustered

It is a 2D representation of the results. To achieve this, the results are transformed to a binary matrix with genes as columns and annotations in rows. For every gene that is part of an annotation a 1 is placed, 0 otherwise. A PCA is later applied to keep as maximun 50 components which later are passed to a t-SNE test (t-distributed Stochastic Neighbor Embedding) to get the final dimensionality reduction.

WordsCloud

In CoAnnotation, the terms appear in the wordscloud according the frequency that each one is reported in the top 50 co-annotated terms. Meanwhile, in the single analysis, the size of the word is related to the -Log(pVal|pValAdj) of the first column that reports it, preferably pValAdj.

Bars-Chart

The bars size represent the -Log(pVal|pValAdj) of the first column that reports it, preferably pValAdj.
